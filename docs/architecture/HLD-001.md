# Architecture: AFib Detection System

**Traces to:** [SRS-001](../software-requirements/SRS-001.md), [SRS-002](../software-requirements/SRS-002.md), [SRS-003](../software-requirements/SRS-003.md)

## Overview

The AFib Detection System is a cloud-based SaaS application that receives ECG data from wearable devices, analyzes it for atrial fibrillation, and presents results to clinicians via a web dashboard.

## System Context

```
┌─────────────┐     ┌─────────────────┐     ┌──────────────┐
│  Wearable   │────▶│ AFib Detection  │────▶│  Clinician   │
│   Device    │     │     System      │     │  Dashboard   │
└─────────────┘     └─────────────────┘     └──────────────┘
                            │
                            ▼
                    ┌───────────────┐
                    │   Database    │
                    └───────────────┘
```

## Component Architecture

### 1. ECG Ingestion Service

- Receives ECG data via secure API
- Validates data format and integrity
- Queues data for processing
- **Implements:** [SRS-001](../software-requirements/SRS-001.md) (performance)

### 2. Detection Engine

- Executes R-R interval analysis algorithm
- Assesses signal quality
- Generates detection results with confidence
- **Implements:** [SRS-002](../software-requirements/SRS-002.md), [SRS-004](../software-requirements/SRS-004.md)

### 3. Persistence Layer

- Stores detection results and metadata
- Maintains audit log
- Provides query interface
- **Implements:** [SRS-005](../software-requirements/SRS-005.md), [SRS-006](../software-requirements/SRS-006.md)

### 4. Clinician Dashboard

- Real-time result display
- Confidence visualization
- Patient management
- **Implements:** [SRS-003](../software-requirements/SRS-003.md)

## Security Architecture

- All data encrypted in transit (TLS 1.3)
- All data encrypted at rest (AES-256)
- Authentication via OAuth 2.0 / OIDC
- Role-based access control

## Deployment

- Cloud provider: AWS
- Container orchestration: Kubernetes
- Database: PostgreSQL with read replicas
