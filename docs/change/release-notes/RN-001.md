# Release Notes: CardioSense v2.1.0

> **Note:** This is an example document with `status: example`. It will appear in the document browser for reference but will not be included in release reviews. Copy this file and change `status` to `draft` to create your own release notes.

## Changes

### New Features

- **Improved AF detection algorithm** — Sensitivity increased from 95% to 97.2% for paroxysmal atrial fibrillation episodes lasting >30 seconds (SRS-001, SRS-002).
- **Real-time alert notifications** — Clinicians receive push notifications within 15 seconds of a confirmed AF episode detection (SRS-005).

### Bug Fixes

- Fixed intermittent false positive alerts during high-motion periods (ANO-003).
- Resolved data export timeout when generating reports >500 pages (ANO-007).

### Regulatory Updates

- Updated IFU to reflect new sensitivity claims per MDR Annex I §23.4.
- Clinical evaluation report updated with post-market clinical follow-up data.

## Known Issues

| ID | Description | Severity | Workaround |
|----|-------------|----------|------------|
| ANO-012 | Battery optimization on Android 14 may delay background sync by up to 60 seconds | Low | Disable battery optimization for the CardioSense app |
| ANO-015 | PDF export may truncate long section titles in landscape mode | Low | Use portrait mode for exports with long section titles |

## Compatibility

- **Mobile App**: iOS 16+, Android 13+
- **Backend API**: v2.1.x (backward compatible with v2.0.x clients)
- **SOUP Dependencies**: See SBOM-001 for updated component list
